# 快速启动和清除数据 - 使用指南

## 📦 已创建的脚本

### 1. 快速启动脚本 (`quick-start.js`)

**命令：**
```bash
npm run quick
```

**功能：**
- ✓ 自动检测服务器是否已在运行
- ✓ 智能启动内存版本服务器
- ✓ 显示访问地址和使用提示
- ✓ 避免重复启动

**使用场景：**
- 首次使用系统
- 快速测试功能
- 演示给他人看

---

### 2. 清除数据脚本 (`clear-data.js`)

**命令：**
```bash
npm run clear
```

**功能：**
- ✓ 自动检测并停止运行中的服务器
- ✓ 释放 3000 端口
- ✓ 清除所有用户数据
- ✓ 需要确认防止误操作
- ✓ 提供下一步操作指引

**使用场景：**
- 重置测试环境
- 清除测试数据
- 解决数据冲突
- 重新开始演示

## ⚠️ 注意事项

### 清除数据脚本

1. **确认提示**
   - 必须输入 `yes` 或 `y` 才能继续
   - 防止意外删除数据

2. **数据丢失**
   - 内存版本：重启后数据自然丢失
   - 清除脚本主要用于停止服务器和释放端口

3. **MongoDB 版本**
   - 脚本会拒绝执行
   - 需要手动清空数据库

### 快速启动脚本

1. **端口检测**
   - 如果 3000 端口被占用会提示
   - 不会强制启动

2. **依赖要求**
   - 确保已运行 `npm install`
   - Node.js >= 14

## 🆘 故障排除

### 问题 1：快速启动失败

**症状：** 提示端口被占用

**解决：**
```bash
# 清除数据（会停止服务器）
npm run clear

# 重新启动
npm run quick
```

### 问题 2：清除数据后无法启动

**症状：** 端口仍在占用

**解决：**
```bash
# Windows - 查找占用端口的进程
netstat -ano | findstr :3000

# 手动停止（替换 PID）
taskkill /F /PID <PID>

# 重新启动
npm run quick
```

### 问题 3：脚本执行错误

**症状：** SyntaxError 或其他错误

**解决：**
```bash
# 检查 Node.js 版本
node --version
# 应该 >= 14

# 重新安装依赖
npm install
```

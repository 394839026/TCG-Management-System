# 更新日志 - 角色权限系统

## 版本 2.0.0 - 角色权限系统

### 新增功能

#### 1. 用户角色系统
- 添加了三种用户角色：user（普通用户）、admin（管理员）、superadmin（超级管理员）
- 新用户注册时默认为 user 角色
- 在 User 模型中添加 `role` 字段

#### 2. 权限控制中间件
- 创建 `authorize()` 中间件函数
- 支持基于角色的路由保护
- 可指定多个允许的角色

#### 3. 管理员功能 API

##### 管理员可用
- `GET /api/auth/users` - 获取所有用户列表
- `POST /api/auth/admin/register` - 创建新用户（只能创建 user 角色）

##### 仅超级管理员可用
- `PUT /api/auth/users/:id/role` - 修改用户角色
- `DELETE /api/auth/users/:id` - 删除用户

#### 4. 前端管理界面

##### 普通用户仪表板
- 显示个人信息（用户名、邮箱、角色）
- 简洁的个人视图

##### 管理员仪表板
- 标签页式界面设计
- **用户列表**：查看所有注册用户
- **创建用户**：表单创建新用户
- 角色徽章显示（不同颜色区分角色）

##### 超级管理员额外功能
- **角色管理**：通过下拉框修改任意用户角色
- 实时保存，立即生效
- 防止修改自己的角色

#### 5. 默认管理员脚本
- `create-admin.js` - 创建初始超级管理员
- 命令：`npm run create-admin`
- 默认凭据：admin@tcg.com / admin123456

### 技术细节

#### 角色验证流程
```
请求 → protect() 验证 token → authorize() 验证角色 → 处理请求
```

#### 权限矩阵
| 功能 | user | admin | superadmin |

| 注册/登录 | ✓ | ✓ | ✓ |

### 安全改进

1. **自我保护机制**
   - 不能修改自己的角色
   - 不能删除自己的账户
   - 防止权限丢失

2. **输入验证**
   - 角色值必须是有效的枚举值
   - 所有输入都经过 express-validator 验证

3. **错误处理**
   - 清晰的错误消息
   - 适当的 HTTP 状态码（401, 403, 404）

### 向后兼容性

**注意**：此更新不向后兼容现有数据库。

如果已有用户数据，需要：
1. 为现有用户手动添加 `role` 字段
2. 或清空数据库重新开始

MongoDB 迁移示例：
```javascript
db.users.updateMany(
  {},
  { $set: { role: 'user' } }
)
```

# 角色权限系统说明

## 角色定义

系统包含三种角色，具有不同的权限级别：

### 1. 普通用户 (user)
- **默认角色**：所有新注册用户默认为普通用户
- **权限**：
  - ✓ 注册账号
  - ✓ 登录系统
  - ✓ 查看个人信息
  - ✗ 不能访问管理功能

### 2. 管理员 (admin)
- **分配方式**：由超级管理员授予
- **权限**：
  - ✓ 普通用户的所有权限
  - ✓ 创建新用户（只能创建普通用户）
  - ✓ 查看所有用户列表
  - ✗ 不能修改用户角色

### 3. 超级管理员 (superadmin)
- **分配方式**：通过脚本初始化或另一个超级管理员授予
- **权限**：
  - ✓ 管理员的所有权限
  - ✓ 创建新用户（可创建普通用户或管理员）
  - ✓ 修改任意用户角色（不能修改自己）
  - ✓ 删除用户（不能删除自己）

## API 端点

### 公开端点（无需认证）

#### 用户注册
```http
POST /api/auth/register
Content-Type: application/json

{
  "username": "newuser",
  "email": "user@example.com",
  "password": "password123"
}
```
**响应**：自动登录并返回 token，角色为 `user`

#### 用户登录
```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```
**响应**：返回用户信息和 token（包含角色）

### 受保护端点（需要认证）

#### 获取当前用户信息
```http
GET /api/auth/me
Authorization: Bearer <token>
```
**权限**：任何登录用户

#### 获取用户列表
```http
GET /api/auth/users
Authorization: Bearer <token>
```
**权限**：admin, superadmin

#### 管理员创建用户
```http
POST /api/auth/admin/register
Authorization: Bearer <token>
Content-Type: application/json

{
  "username": "newuser",
  "email": "new@example.com",
  "password": "password123",
  "role": "user"  // 仅superadmin可指定角色
}
```
**权限**：admin, superadmin
**注意**：普通管理员只能创建 user 角色

#### 修改用户角色
```http
PUT /api/auth/users/:id/role
Authorization: Bearer <token>
Content-Type: application/json

{
  "role": "admin"  // user, admin, superadmin
}
```
**权限**：superadmin only
**限制**：不能修改自己的角色

#### 删除用户
```http
DELETE /api/auth/users/:id
Authorization: Bearer <token>
```
**权限**：superadmin only
**限制**：不能删除自己

## 快速开始

### 1. 创建超级管理员

使用 MongoDB 时：
```bash
npm run create-admin
```

这将创建默认的超级管理员：
- 用户名：admin
- 邮箱：admin@tcg.com
- 密码：admin123456

### 2. 启动服务器

内存版本（演示用）：
```bash
npm run demo
```

MongoDB 版本（生产用）：
```bash
npm start
# 或开发模式
npm run dev
```

### 3. 访问系统

打开浏览器访问：http://localhost:3000

## 前端界面功能

### 普通用户界面
- 注册/登录表单
- 个人信息仪表板
- 显示用户名、邮箱、角色

### 管理员界面
- 所有普通用户功能
- **用户列表标签页**：查看所有用户
- **创建用户标签页**：创建新用户

### 超级管理员界面
- 所有管理员功能
- **角色管理标签页**：修改任意用户角色
  - 下拉选择框更改角色
  - 实时保存

## 安全特性

1. **密码加密**：使用 bcrypt 进行哈希
2. **JWT 认证**：7天有效期的令牌
3. **角色验证**：中间件级别权限检查
4. **输入验证**：express-validator 验证所有输入
5. **自我保护**：防止修改/删除自己的账户

## 测试场景

### 场景 1：普通用户尝试访问管理功能
```
1. 以普通用户登录
2. 尝试访问 /api/auth/users
结果：403 Forbidden - 需要 admin 或 superadmin 权限
```

### 场景 2：管理员创建用户
```
1. 以管理员登录
2. 进入"创建用户"标签页
3. 填写用户信息
4. 提交
结果：创建成功，新用户角色为 user
```

### 场景 3：超级管理员提升用户权限
```
1. 以超级管理员登录
2. 进入"角色管理"标签页
3. 找到目标用户
4. 从下拉框选择新角色（admin 或 superadmin）
5. 确认修改
结果：用户角色更新成功
```

## 数据库结构

### User 模型
```javascript
{
  username: String (3-20字符，唯一),
  email: String (唯一，邮箱格式),
  password: String (bcrypt加密，最少6字符),
  role: String (enum: 'user', 'admin', 'superadmin'),
  createdAt: Date
}
```

## 常见问题

**Q: 如何创建第一个超级管理员？**
A: 运行 `npm run create-admin`（需要 MongoDB）

**Q: 普通管理员能创建管理员吗？**
A: 不能，只有超级管理员可以指定角色

**Q: 超级管理员能把自己降级吗？**
A: 不能，系统防止修改自己的角色

**Q: 内存版本和MongoDB版本有什么区别？**
A: 功能完全相同，只是数据存储方式不同。内存版本重启后数据丢失。

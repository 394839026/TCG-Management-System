## 1. 系统概述

### 1.1 项目背景

TCG卡牌综合管理系统是一个功能完整的卡牌游戏管理平台，支持用户进行卡牌收集、交易、组队、抽卡等核心游戏功能。系统采用现代化Web技术栈构建，具有高性能、高可用性和良好的扩展性。

### 1.2 核心特性

- ✅ 用户认证与权限管理（三级权限体系）
- ✅ 个人库存与团队库存管理
- ✅ 平台商店与兑换系统（积分/星币双货币）
- ✅ 抽卡模拟器与概率配置
- ✅ 抽卡记录与收集进度追踪
- ✅ 店铺与订单系统
- ✅ 组队与好友系统
- ✅ 卡组构建与管理
- ✅ 任务系统（每日/每周/成就）
- ✅ 公告系统
- ✅ 群聊功能
- ✅ 经验值与等级系统
- ✅ 高并发优化机制

### 1.3 用户角色

| 角色 | 说明 | 权限级别 |
|------|------|----------|
| 普通用户 (user) | 可以使用所有基础功能 | 1级 |
| 管理员 (admin) | 可以管理商品、任务、公告等 | 2级 |
| 超级管理员 (superadmin) | 拥有所有权限 | 3级 |
| 店铺店主 | 可以管理自己的店铺、商品、订单 | 店铺权限 |
| 战队队长 | 可以管理战队成员、战队库存 | 战队权限 |

### 1.4 支持的游戏类型

- 符文战场 (Rune)
- 数码宝贝卡牌游戏 (Digimon)
- 宝可梦卡牌游戏 (Pokemon)
- 影之诗进化版 (Shadowverse Evolve)

### 2.2 架构模式

系统采用前后端分离架构（SPA + RESTful API）：

- **前端**: 单页面应用（SPA），React + TypeScript
- **后端**: RESTful API服务，Express.js + MongoDB
- **通信**: JSON格式，JWT令牌认证
- **部署**: 前后端分离部署，支持反向代理

#### 兑换流程（高并发安全）

```
1. 获取分布式锁 (key: redeem:userId:itemId)
     │
     ▼
2. 开始MongoDB事务
     │
     ▼
3. 查询商品信息，验证有效性
     │
     ▼
4. 验证:
   - 商品是否激活
   - 是否在有效期
   - 库存是否足够
   - 用户是否超限购
   - 用户余额是否足够
     │
     ▼
5. 原子扣减用户余额 ($inc: {points/coins: -totalCost})
     │
     ▼
6. 发放物品（如果是实物）
     │
     ▼
7. 原子增加商品已兑换数 ($inc: {redeemedCount: quantity})
     │
     ▼
8. 创建兑换记录
     │
     ▼
9. 提交事务
     │
     ▼
10. 释放锁
     │
     ▼
11. 返回成功结果
```

```javascript
  success: false,
```

### 9.3 高并发优化

#### 优化架构图

```

    │
    ▼

│  1. 限流层 (Rate Limiter)       │

    │
    ▼
┌─────────────────────────────────┐
│  2. 分布式锁层 (Lock Manager)   │
│  - 防止用户重复提交            │
│  - 防止竞态条件                │
│  - 自动超时清理 (30秒)         │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  3. 事务层 (Transaction)        │
│  - MongoDB原子事务             │
│  - 全部成功或全部回滚          │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  4. 原子操作层 (Atomic $inc)   │

│  - 原子库存更新               │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  5. 连接池层 (Connection Pool) │
│  - 5-20个连接复用             │
```

### 11.5 常见问题

#### Q: 如何重置所有数据？
A: 内存版本重启即可；MongoDB版本需要清空数据库。

#### Q: 忘记超级管理员密码怎么办？
A: 运行 `npm run setup-superadmin` 重新创建。

#### Q: 可以自定义角色名称吗？
A: 可以，修改User模型中的enum值和相关代码。

#### Q: 支持OAuth登录吗？
A: 目前不支持，可以后续添加Google、GitHub等OAuth。

#### Q: 如何备份数据？
A: MongoDB版本使用mongodump；内存版本不支持持久化。

#### Q: 系统支持哪些卡牌游戏？
A: 目前支持符文战场、数码宝贝、宝可梦、影之诗进化版，可扩展其他游戏。

#### Q: 如何进行Excel批量导入？
A: 在库存页面点击"导入Excel"按钮，按照模板格式上传文件。

# 角色权限系统 - 快速测试指南

## 方法一：使用前端界面（推荐）

### 步骤 1：启动服务器
```bash
npm run demo
```

### 步骤 2：打开浏览器
访问 http://localhost:3000

### 步骤 3：注册普通用户
1. 点击"立即注册"
2. 填写信息：
   - 用户名：testuser
   - 邮箱：test@example.com
   - 密码：password123
3. 点击"注册"
4. **预期结果**：登录成功，显示普通用户仪表板

### 步骤 4：退出登录
点击"退出登录"按钮

### 步骤 5：创建管理员账户（需要后端操作）

由于前端注册只能创建普通用户，我们需要通过 API 创建管理员。

使用 curl 或 Postman：

```bash
# 先注册一个用户
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","email":"admin@test.com","password":"admin123"}'

# 获取返回的 token
```

然后手动在数据库中将该用户角色改为 admin 或 superadmin（仅用于测试）。

### 步骤 6：以管理员身份登录
1. 使用管理员凭据登录
2. **预期结果**：显示管理面板
3. 应该看到"用户列表"和"创建用户"标签页

### 步骤 7：测试创建用户
1. 点击"创建用户"标签页
2. 填写新用户信息
3. 点击"创建用户"
4. **预期结果**：创建成功，用户出现在用户列表中

## 方法二：使用 API 测试（完整测试）

### 准备工作
```bash
# 确保服务器正在运行
npm run demo
```

### 测试 1：普通用户注册
```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "test@example.com",
    "password": "password123"
  }'
```

**预期响应**：
```json
{
  "success": true,
  "data": {
    "_id": 1,
    "username": "testuser",
    "email": "test@example.com",
    "role": "user",
    "token": "..."
  }
}
```

### 测试 2：普通用户尝试访问管理功能
```bash
# 使用上一步返回的 token
curl http://localhost:3000/api/auth/users \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

**预期响应**：
```json
{
  "message": "需要以下权限之一: admin, superadmin"
}
```
状态码：403

### 测试 3：管理员创建用户
```bash
# 首先需要一个管理员 token
curl -X POST http://localhost:3000/api/auth/admin/register \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "newuser",
    "email": "new@example.com",
    "password": "password123"
  }'
```

**预期响应**：
```json
{
  "success": true,
  "message": "用户创建成功",
  "data": {
    "_id": 2,
    "username": "newuser",
    "email": "new@example.com",
    "role": "user"
  }
}
```

### 测试 4：获取用户列表
```bash
curl http://localhost:3000/api/auth/users \
  -H "Authorization: Bearer ADMIN_TOKEN"
```

**预期响应**：
```json
{
  "success": true,
  "count": 2,
  "data": [
    {
      "id": 1,
      "username": "testuser",
      "email": "test@example.com",
      "role": "user"
    },
    {
      "id": 2,
      "username": "newuser",
      "email": "new@example.com",
      "role": "user"
    }
  ]
}
```

### 测试 5：修改用户角色（超级管理员）
```bash
curl -X PUT http://localhost:3000/api/auth/users/1/role \
  -H "Authorization: Bearer SUPERADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"role": "admin"}'
```

**预期响应**：
```json
{
  "success": true,
  "message": "用户角色更新成功",
  "data": {
    "_id": 1,
    "username": "testuser",
    "email": "test@example.com",
    "role": "admin"
  }
}
```

## 常见问题排查

### 问题 1：无法连接到服务器
**解决**：
```bash
# 检查服务器是否运行
# 查看终端输出是否有 "服务器运行在 http://localhost:3000"

# 重新启动服务器
npm run demo
```

### 问题 2：注册时提示用户名已存在
**解决**：使用不同的用户名或邮箱，因为内存版本重启后数据会保留在当前会话中

### 问题 3：403 Forbidden 错误
**原因**：当前用户没有足够权限
**解决**：
- 检查用户的角色
- 确保使用正确的 token
- 某些操作需要 admin 或 superadmin 权限

### 问题 4：前端不显示管理界面
**原因**：用户角色不是 admin 或 superadmin
**解决**：
1. 检查登录用户的 role 字段
2. 需要通过 API 或数据库手动设置角色

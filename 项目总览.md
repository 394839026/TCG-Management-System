# TCG 卡牌综合管理系统 - 用户认证与权限管理

## 项目概述

这是一个完整的用户认证和权限管理系统，支持三种角色层级，提供现代化的 Web 界面和 RESTful API。

## 核心功能

### 1. 用户认证
- ✓ 用户注册（自动加密密码）
- ✓ 用户登录（JWT 令牌）
- ✓ 会话管理
- ✓ 自动登录（localStorage）

### 2. 角色权限系统
- **普通用户 (user)**：基本访问权限
- **管理员 (admin)**：用户管理权限
- **超级管理员 (superadmin)**：完全控制权限

### 3. 管理功能
- 查看用户列表
- 创建新用户
- 修改用户角色
- 删除用户

### 4. 现代化界面
- 响应式设计（支持移动端）
- 渐变紫色主题
- 标签页式管理界面
- 实时数据更新

## 技术栈

| 类别 | 技术 |
|------|------|
| 后端 | Node.js, Express.js |
| 数据库 | MongoDB (Mongoose) / 内存存储 |
| 认证 | JWT (jsonwebtoken) |
| 加密 | bcryptjs |
| 验证 | express-validator |
| 前端 | HTML5, CSS3, Vanilla JavaScript |

## API 端点

### 公开端点
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/auth/register | 用户注册 |
| POST | /api/auth/login | 用户登录 |

### 受保护端点
| 方法 | 路径 | 权限 | 描述 |
|------|------|------|------|
| GET | /api/auth/me | 任何用户 | 获取当前用户信息 |
| GET | /api/auth/users | admin+ | 获取所有用户 |
| POST | /api/auth/admin/register | admin+ | 创建新用户 |
| PUT | /api/auth/users/:id/role | superadmin | 修改用户角色 |
| DELETE | /api/auth/users/:id | superadmin | 删除用户 |
| GET | /api/dashboard | 任何用户 | 访问仪表板 |

## 安全特性

1. **密码安全**
   - bcrypt 加密（10轮盐值）
   - 最小长度 6 字符
   - 永不存储明文密码

2. **认证安全**
   - JWT 令牌（7天有效期）
   - Bearer token 认证
   - 令牌过期自动失效

3. **授权安全**
   - 基于角色的访问控制（RBAC）
   - 中间件级别权限检查
   - 防止越权操作

4. **输入验证**
   - express-validator 验证
   - SQL 注入防护
   - XSS 防护

5. **自我保护**
   - 不能修改自己的角色
   - 不能删除自己的账户
   - 防止权限丢失

## 常见问题

**Q: 如何重置所有数据？**
A: 内存版本重启即可；MongoDB 版本需要清空数据库

**Q: 忘记超级管理员密码怎么办？**
A: 运行 `npm run create-admin` 重新创建（MongoDB 版本）

**Q: 可以自定义角色名称吗？**
A: 可以，修改 User 模型中的 enum 值和相关代码

**Q: 支持 OAuth 登录吗？**
A: 目前不支持，可以后续添加 Google、GitHub 等 OAuth

**Q: 如何备份数据？**
A: MongoDB 版本使用 mongodump；内存版本不支持持久化

# 库存模块权限更新说明

## 更新日期
2026-04-28

## 更新内容

### 权限调整

根据新的需求，对库存模块的权限进行了以下调整：

#### 普通用户 (user)
**之前的权限：**
- ✓ 添加物品
- ✓ 编辑物品的所有字段
- ✓ 删除物品
- ✓ 查看自己的库存

**现在的权限：**
- ✗ **不能**添加新物品
- ✓ 只能修改物品的**数量**字段
- ✓ 可以删除自己的物品
- ✓ 可以查看自己的库存

#### 管理员 (admin) 和 超级管理员 (superadmin)
- ✓ 完整的添加、编辑、删除权限
- ✓ 可以查看所有用户的库存
- ✓ 可以修改任意字段

## 技术实现

### 后端变更

#### 1. 添加物品路由 (`POST /api/inventory`)
```javascript
// 添加了 authorize 中间件，只允许 admin 和 superadmin
router.post('/',
  protect,
  authorize('admin', 'superadmin'),  // 新增
  ...
);
```

#### 2. 更新物品路由 (`PUT /api/inventory/:id`)
```javascript
// 普通用户只能修改数量字段
if (req.user.role === 'user') {
  // 检查是否只修改了数量字段
  const allowedFields = ['quantity'];
  const requestedFields = Object.keys(req.body);
  const hasInvalidFields = requestedFields.some(field => !allowedFields.includes(field));
  
  if (hasInvalidFields) {
    return res.status(403).json({ message: '普通用户只能修改数量' });
  }
}
```

### 前端变更

#### 1. 隐藏添加按钮
```javascript
// 在 showInventoryPage() 函数中
const user = JSON.parse(localStorage.getItem('user'));
const addButton = document.querySelector('.inventory-actions button:first-child');
if (user.role === 'user') {
  addButton.style.display = 'none';  // 隐藏添加按钮
}
```

#### 2. 编辑模态框动态显示
```javascript
// 普通用户只显示数量字段
if (user.role === 'user') {
  document.querySelector('#editItemModal h3').textContent = '修改数量';
  document.getElementById('edit-item-name').parentElement.style.display = 'none';
  document.getElementById('edit-item-value').parentElement.parentElement.style.display = 'none';
}
```

#### 3. 提交数据过滤
```javascript
// 根据角色发送不同的数据
if (user.role === 'user') {
  updateData = { quantity: parseInt(...) };
} else {
  updateData = {
    itemName: ...,
    quantity: ...,
    value: ...
  };
}
```

## 用户体验变化

### 普通用户视角

**之前：**
1. 可以看到"添加物品"按钮
2. 点击编辑可以看到所有字段
3. 可以修改名称、类型、数量、价格等

**现在：**
1. "添加物品"按钮被隐藏
2. 点击编辑只显示"数量"字段
3. 标题变为"修改数量"
4. 尝试修改其他字段会收到错误提示

### 管理员视角

没有任何变化，仍然拥有完整的管理权限。

## API 行为变化

### 普通用户尝试添加物品
```http
POST /api/inventory
Authorization: Bearer user_token
Content-Type: application/json

{
  "itemName": "测试卡片",
  "quantity": 5
}
```

**响应：**
```json
{
  "message": "需要以下权限之一: admin, superadmin"
}
```
**状态码：** 403 Forbidden

### 普通用户尝试修改非数量字段
```http
PUT /api/inventory/123
Authorization: Bearer user_token
Content-Type: application/json

{
  "itemName": "新名称",
  "quantity": 10
}
```

**响应：**
```json
{
  "message": "普通用户只能修改数量"
}
```
**状态码：** 403 Forbidden

### 普通用户正确修改数量
```http
PUT /api/inventory/123
Authorization: Bearer user_token
Content-Type: application/json

{
  "quantity": 10
}
```

**响应：**
```json
{
  "success": true,
  "message": "物品更新成功",
  "data": { ... }
}
```
**状态码：** 200 OK

## 使用场景

### 场景 1：普通用户更新库存数量

用户获得了新的卡牌，需要更新数量：

1. 登录系统
2. 进入"我的库存"
3. 找到对应物品
4. 点击"编辑"按钮
5. 在弹出的窗口中修改数量
6. 点击"保存"

### 场景 2：普通用户需要添加新物品

用户有新的物品类型需要录入：

1. 联系管理员
2. 提供物品信息
3. 管理员代为添加

或者：

1. 申请提升为管理员角色
2. 获得添加权限

### 场景 3：管理员批量添加物品

管理员为用户添加初始库存：

1. 以管理员身份登录
2. 进入"库存管理"
3. 点击"添加物品"
4. 填写完整信息
5. 可以为不同用户添加物品（通过API）

## 注意事项

1. **数据一致性**
   - 普通用户无法添加新物品，确保库存由管理员统一管理
   - 避免重复物品和混乱的分类

2. **权限控制**
   - 后端有严格的字段验证
   - 即使绕过前端限制，后端也会拒绝非法请求

3. **用户体验**
   - 普通用户界面更简洁
   - 减少了误操作的可能性
   - 专注于核心的数量管理功能

4. **管理流程**
   - 建议建立物品添加审核流程
   - 管理员定期检查库存数据
   - 保持物品信息的规范性

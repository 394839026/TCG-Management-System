## 🗑️ 清除数据脚本

### 使用方法

```bash
npm run clear
```

### 功能特点

1. **安全检查**
   - 检测 MongoDB 配置
   - 防止误操作生产环境

2. **确认提示**
   - 需要输入 yes 确认
   - 防止意外删除

3. **自动清理**
   - 停止运行中的服务器
   - 释放端口
   - 清除所有用户数据

4. **详细反馈**
   - 显示清理进度
   - 提供下一步操作指引

### 使用场景

- ✓ 重置测试环境

## 📋 完整工作流程

### 场景 1：首次使用

```bash
# 1. 快速启动
npm run quick

# 2. 打开浏览器注册第一个用户（自动成为超级管理员）
# http://localhost:3000

# 3. 开始使用系统
```

### 场景 2：重置系统

```bash
# 1. 清除所有数据
npm run clear
# 输入 yes 确认

# 2. 重新启动
npm run quick

# 3. 重新注册超级管理员
```

### 场景 3：日常开发

```bash
# 启动服务器
npm run dev      # 开发模式（MongoDB，自动重启）
# 或
npm run demo     # 演示模式（内存存储）
# 或
npm run quick    # 快速启动（智能检测）
```

## ⚠️ 注意事项

### 清除数据脚本

1. **内存版本**
   - 清除操作会停止服务器
   - 所有用户数据丢失
   - 需要重新启动

2. **MongoDB 版本**
   - 脚本会拒绝执行
   - 需要手动清空数据库
   - 使用 mongosh 命令

3. **数据安全**
   - 生产环境慎用
   - 建议先备份数据
   - 确认后再操作

### 快速启动脚本

1. **端口占用**
   - 如果 3000 端口被占用会提示
   - 需要先停止现有服务

2. **依赖检查**
   - 确保已运行 `npm install`
   - Node.js 版本 >= 14

## 🆘 常见问题

**Q: 快速启动失败怎么办？**
A: 检查是否有其他程序占用 3000 端口，或手动运行 `npm run demo`

**Q: 清除数据后无法启动？**
A: 确保完全停止了之前的服务器，等待几秒后再启动

**Q: 如何只清除特定用户？**
A: 需要使用 MongoDB shell 或管理界面进行操作

**Q: 可以在生产环境使用吗？**
A: `npm run quick` 可以，但 `npm run clear` 请谨慎使用

# 库存模块使用指南

## 概述

库存模块允许用户管理自己的TCG卡牌收藏，管理员可以查看他人库存，超级管理员可以修改他人库存。

## 权限说明

| 功能 | 普通用户 | 管理员 | 超级管理员 |
|------|---------|--------|-----------|
| 查看自己的库存 | ✓ | ✓ | ✓ |
| 添加物品 | ✓ | ✓ | ✓ |
| 编辑自己的物品 | ✓ | ✓ | ✓ |
| 删除自己的物品 | ✓ | ✓ | ✓ |
| 查看他人库存 | ✗ | ✓ | ✓ |
| 编辑他人物品 | ✗ | ✓ | ✓ |
| 删除他人物品 | ✗ | ✓ | ✓ |

## API 端点

### 个人库存管理

#### 获取我的库存列表
```http
GET /api/inventory
Authorization: Bearer <token>
```

查询参数：
- `itemType`: 过滤类型 (card, booster, box, accessory, other)
- `search`: 搜索关键词
- `sort`: 排序字段 (默认 createdAt)
- `order`: 排序方向 (asc/desc)

#### 获取库存统计
```http
GET /api/inventory/stats
Authorization: Bearer <token>
```

返回：
- 按类型分组的统计
- 总物品数量
- 总价值

#### 添加物品
```http
POST /api/inventory
Authorization: Bearer <token>
Content-Type: application/json

{
  "itemName": "青眼白龙",
  "itemType": "card",
  "quantity": 3,
  "condition": "near_mint",
  "value": 150.00,
  "description": "第一版",
  "tags": ["rare", "dragon"]
}
```

#### 更新物品
```http
PUT /api/inventory/:id
Authorization: Bearer <token>
Content-Type: application/json

{
  "quantity": 5,
  "value": 200.00
}
```

#### 删除物品
```http
DELETE /api/inventory/:id
Authorization: Bearer <token>
```

### 管理员功能

#### 查看指定用户库存
```http
GET /api/inventory/admin/users/:userId
Authorization: Bearer <admin_token>
```

## 物品类型

- `card` - 卡牌
- `booster` - 补充包
- `box` - 盒装
- `accessory` - 配件（卡套、卡盒等）
- `other` - 其他

## 物品状态

- `mint` - 完美
- `near_mint` - 近完美
- `excellent` - 极佳
- `good` - 良好
- `fair` - 一般
- `poor` - 较差

## 前端使用

### 访问库存页面

1. 登录后，在仪表板点击"我的库存"或"库存管理"按钮
2. 进入库存管理页面

### 添加物品

1. 点击"添加物品"按钮
2. 填写物品信息：
   - 物品名称（必填）
   - 类型
   - 数量（必填）
   - 状态
   - 单价
   - 描述
3. 点击"保存"

### 编辑物品

1. 在物品列表中点击"编辑"按钮
2. 修改信息
3. 点击"保存"

### 删除物品

1. 在物品列表中点击"删除"按钮
2. 确认删除

### 查看统计

页面顶部显示：
- 总物品数
- 总价值
- 物品种类数

## 使用示例

### 场景 1：普通用户管理自己的收藏

```bash
# 1. 登录获取 token
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@test.com","password":"password123"}'

# 2. 添加一张卡牌
curl -X POST http://localhost:3000/api/inventory \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "itemName": "青眼白龙",
    "itemType": "card",
    "quantity": 1,
    "condition": "mint",
    "value": 500
  }'

# 3. 查看我的库存
curl http://localhost:3000/api/inventory \
  -H "Authorization: Bearer YOUR_TOKEN"

# 4. 查看统计
curl http://localhost:3000/api/inventory/stats \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 场景 2：管理员查看用户库存

```bash
# 1. 获取用户列表
curl http://localhost:3000/api/auth/users \
  -H "Authorization: Bearer ADMIN_TOKEN"

# 2. 查看指定用户的库存
curl http://localhost:3000/api/inventory/admin/users/USER_ID \
  -H "Authorization: Bearer ADMIN_TOKEN"
```

### 场景 3：管理员修改他人物品

```bash
# 修改指定物品
curl -X PUT http://localhost:3000/api/inventory/ITEM_ID \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"quantity": 10}'
```

## 数据模型

```javascript
{
  userId: ObjectId,          // 所属用户ID
  itemName: String,          // 物品名称
  itemType: String,          // 类型枚举
  quantity: Number,          // 数量
  condition: String,         // 状态枚举
  value: Number,             // 单价
  description: String,       // 描述
  tags: [String],            // 标签数组
  createdAt: Date,           // 创建时间
  updatedAt: Date            // 更新时间
}
```

## 注意事项

1. **权限控制**
   - 普通用户只能操作自己的物品
   - 管理员和超级管理员可以操作所有物品

2. **数据验证**
   - 物品名称不能为空
   - 数量必须是非负整数
   - 价值不能为负数

3. **内存版本**
   - 演示模式下数据存储在内存中
   - 重启服务器后数据丢失

4. **MongoDB 版本**
   - 数据持久化存储
   - 支持更复杂的查询和聚合

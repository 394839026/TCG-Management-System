# 高并发优化说明

本系统已经针对高并发场景进行了全面的优化，确保在大规模用户同时操作时的数据一致性和系统稳定性。

## 优化概述

### 1. 数据库连接池优化 (config/db.js)

**问题**: 无限制的数据库连接会导致资源耗尽
**解决方案**: 使用 MongoDB 连接池配置

```javascript
// 连接池配置
const poolConfig = {
  poolSize: 10,           // 基础连接池大小
  maxPoolSize: 20,        // 最大连接数
  minPoolSize: 5,         // 最小连接数
  socketTimeoutMS: 45000, // Socket超时
  connectTimeoutMS: 30000,// 连接超时
};
```

**特性**:
- 连接复用，避免频繁创建/销毁连接
- 自动连接管理
- 心跳机制确保连接可用性
- 优雅的关闭处理

### 2. 读写优化 (config/db.js)

**配置**:
```javascript
readConcern: { level: 'majority' }, // 读取大多数确认的数据
writeConcern: { w: 'majority', j: true, wtimeout: 5000 }, // 写入确认
readPreference: 'primaryPreferred',   // 主优先读取策略
retryWrites: true,                    // 自动重试写入
retryReads: true,                     // 自动重试读取
```

**作用**:
- 确保数据一致性
- 提高读写可用性
- 自动故障恢复

### 3. 分布式锁机制 (middleware/concurrencyControl.js)

**应用场景**: 防止同一用户重复兑换、防止超卖

```javascript
withLock(`redeem:${userId}:${storeItemId}`, async () => {
  // 临界区代码，确保串行执行
})
```

**特性**:
- 支持内存锁（开发环境）
- 支持分布式锁（生产环境，基于 MongoDB）
- 自动重试获取锁
- 自动清理过期锁
- 30秒超时保护

### 4. 事务机制 (middleware/concurrencyControl.js)

**应用场景**: 涉及多表操作的场景（如：扣钱 + 发货 + 记录）

```javascript
const result = await executeTransaction(async (session) => {
  // 所有操作使用同一个 session
  await Model1.create([data], { session });
  await Model2.findByIdAndUpdate(id, updateData, { new: true, session });
});
```

**作用**:
- 原子性：全部成功或全部失败
- 一致性：确保数据状态一致
- 隔离性：避免并发干扰
- 持久性：确保提交后持久化

### 5. 原子操作 ($inc)

**应用场景**: 库存扣减、余额增减

**优化前**:
```javascript
// ❌ 有并发安全问题
user.coins -= amount;
await user.save();
```

**优化后**:
```javascript
// ✅ 使用原子操作
const updatedUser = await User.findByIdAndUpdate(
  userId,
  { $inc: { coins: -amount } },
  { new: true, session }
);
```

**优势**:
- MongoDB 原子性保证
- 无竞态条件
- 性能更好（一次查询 + 更新）

### 6. 限流机制 (middleware/concurrencyControl.js)

**速率限制器**:
```javascript
const gachaLimiter = new RateLimiter(10, 60000);  // 每分钟10次抽卡
const redeemLimiter = new RateLimiter(5, 60000);  // 每分钟5次兑换
const generalLimiter = new RateLimiter(100, 60000); // 每分钟100次请求
```

**作用**:
- 防止恶意刷屏
- 防止 DDoS 攻击
- 保护系统资源

### 7. 数据库索引优化 (server.js - ensureIndexes)

**已创建的关键索引**:

**平台商店**:
```javascript
// 快速查询有效商品

// 排序优化

```

**兑换记录**:
```javascript

```

**抽卡记录**:
```javascript

```

**用户数据**:


## 关键路由优化

### 1. 平台商店兑换 (routes/platformStore.js)

**安全措施**:
- ✅ 限流保护（每分钟5次）
- ✅ 分布式锁（防止重复兑换）
- ✅ 事务包裹（所有操作原子性）
- ✅ 原子操作（扣钱 + 增加已兑换数）
- ✅ 库存检查
- ✅ 用户限购检查

**执行流程**:
1. 获取锁 -> 防止用户重复提交
2. 开始事务 -> 确保原子性
3. 检查商品状态 -> 下架/过期拦截
4. 检查库存 -> 防止超卖
5. 检查用户限购 -> 防刷
6. 扣钱（原子操作）
7. 发货（如果是实物）
8. 更新已兑换数（原子操作）
9. 记录兑换日志
10. 提交事务 -> 全部成功或全部回滚
11. 释放锁

### 2. 抽卡消耗 (routes/gacha.js)

**安全措施**:
- ✅ 限流保护（每分钟10次）
- ✅ 分布式锁
- ✅ 事务包裹
- ✅ 原子操作扣钱

### 3. 每日礼物 (routes/gacha.js)

**安全措施**:
- ✅ 限流保护
- ✅ 分布式锁
- ✅ 事务包裹
- ✅ 日期检查防止重复领取

## 性能优化建议

1. **MongoDB 配置**:
   - 使用副本集（3节点以上）
   - 配置读写分离
   - 启用监控（MongoDB Compass）

## 故障处理

### 1. 锁超时
- 自动重试获取锁
- 3次重试失败返回错误

### 2. 事务冲突
- MongoDB 自动处理冲突
- 冲突时返回错误，让用户重试

### 3. 数据库连接失败
- 连接池自动重连
- 保留一定数量的备用连接

### 4. 限流触发
- 友好提示用户稍后重试
- 可根据需要调整限流阈值

## 总结

本系统通过多层防御确保高并发安全:
1. 限流层 -> 保护系统资源
2. 锁机制层 -> 防止竞态条件
3. 事务层 -> 保证数据一致性
4. 原子操作层 -> 高效安全的数值更新
5. 索引层 -> 提升查询性能
6. 连接池层 -> 高效利用数据库资源

这套机制可以有效应对:
- 用户恶意刷屏
- 秒杀/抢购活动
- 日常高并发访问
- 防止超卖/超扣

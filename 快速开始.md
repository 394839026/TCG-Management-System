# 快速开始指南

## 🚀 立即体验（无需 MongoDB）

系统提供了内存存储的演示版本，可以立即体验所有功能：

```bash
npm run demo
```

然后在浏览器中访问: http://localhost:3000

**注意**: 内存版本重启后数据会丢失，仅用于演示和测试。

## 📦 完整安装（使用 MongoDB）

### 步骤 1: 设置 MongoDB

选择以下任一方式：

**选项 A: MongoDB Atlas（云端 - 推荐）**
1. 访问 https://www.mongodb.com/cloud/atlas/register
2. 创建免费账户和集群
3. 获取连接字符串并更新 `.env` 文件中的 `MONGODB_URI`

**选项 B: 本地安装**
- Windows: 从 https://www.mongodb.com/try/download/community 下载安装
- macOS: `brew install mongodb-community`
- Linux: `sudo apt install mongodb`

详见 [MONGODB_SETUP.md](MONGODB_SETUP.md)

### 步骤 2: 启动服务器

```bash
npm start
# 或开发模式（自动重启）
npm run dev
```

### 步骤 3: 访问应用

打开浏览器访问: http://localhost:3000

## ✨ 功能演示

### 1. 用户注册
- 点击"立即注册"
- 填写用户名、邮箱和密码
- 点击注册按钮

### 2. 用户登录
- 输入邮箱和密码
- 点击登录按钮
- 成功后会自动跳转到仪表板

### 3. 受保护的内容
- 登录后可以看到仪表板
- 显示用户信息和欢迎消息
- 只有登录用户才能访问

### 4. 退出登录
- 点击"退出登录"按钮
- 返回登录页面

## 📝 API 端点

| 方法 | 路径 | 描述 | 认证 |
|------|------|------|------|
| POST | /api/auth/register | 用户注册 | 否 |
| POST | /api/auth/login | 用户登录 | 否 |
| GET | /api/auth/me | 获取当前用户 | 是 |
| GET | /api/dashboard | 仪表板 | 是 |

## 🔐 安全特性

- ✅ 密码使用 bcrypt 加密（10轮盐值）
- ✅ JWT 令牌认证（7天有效期）
- ✅ 输入验证和清理
- ✅ CORS 支持
- ✅ 防止 SQL 注入

## 🛠️ 技术栈

- **后端**: Node.js + Express.js
- **数据库**: MongoDB + Mongoose（或内存存储）
- **认证**: JWT (JSON Web Tokens)
- **密码加密**: bcryptjs
- **验证**: express-validator
- **前端**: HTML5 + CSS3 + Vanilla JavaScript

## ❓ 常见问题

**Q: 为什么我无法连接到 MongoDB？**
A: 确保 MongoDB 服务正在运行，或者使用内存版本 `npm run demo`

**Q: 如何修改 JWT 密钥？**
A: 编辑 `.env` 文件中的 `JWT_SECRET`

**Q: 如何更改端口？**
A: 编辑 `.env` 文件中的 `PORT`，默认是 3000

**Q: 数据保存在哪里？**
A: MongoDB 版本数据保存在数据库中；内存版本重启后数据丢失

## 📞 支持

如有问题，请检查：
1. Node.js 是否已安装（v14+）
2. MongoDB 是否正在运行（如使用完整版本）
3. `.env` 文件配置是否正确
4. 控制台是否有错误信息

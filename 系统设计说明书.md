## 1. 系统概述

### 1.1 项目背景

TCG（Trading Card Game）卡牌综合管理系统是一个功能完整的卡牌游戏管理平台，支持用户进行卡牌收集、交易、组队、抽卡等核心游戏功能。系统采用现代 Web 技术栈构建，具有高性能、高可用性和良好的扩展性。

### 1.2 核心特性

- ✅ 用户认证与权限管理
- ✅ 个人库存与团队库存管理
- ✅ 平台商店与兑换系统
- ✅ 抽卡模拟器与概率配置
- ✅ 抽卡记录与收集进度
- ✅ 店铺与交易系统
- ✅ 组队与好友系统
- ✅ 任务系统
- ✅ 公告系统
- ✅ 群聊功能
- ✅ 经验值与等级系统

### 1.3 用户角色

| 角色 | 说明 |
|-----|-----|
| 普通用户 | 可以使用所有基础功能 |
| 管理员 | 可以管理商品、任务、公告、概率配置等 |
| 超级管理员 | 拥有所有权限，包括用户管理、权限管理、库存模板管理等 |
| 店铺店主 | 可以管理自己的店铺、商品、订单、消息 |

#### 兑换流程（高并发安全）

```
1. 获取分布式锁 (key: redeem:userId:itemId)
     │
     ▼
2. 开始 MongoDB 事务
     │
     ▼
3. 查询商品信息
     │
     ▼
4. 验证：
   - 商品是否激活
   - 是否在有效期
   - 库存是否足够
   - 用户是否超限购
     │
     ▼
5. 原子扣减用户余额
   ($inc: { coins: -totalCost })
     │
     ▼
6. 发放物品（如果是实物）
     │
     ▼
7. 原子增加商品已兑换数
   ($inc: { redeemedCount: quantity })
     │
     ▼
8. 创建兑换记录
     │
     ▼
9. 提交事务
     │
     ▼
10. 释放锁
     │
     ▼
11. 返回成功结果
```

## 4. 数据库设计

### 4.1 核心数据模型

#### 用户模型 (User)
```javascript
{
  username: String,       // 用户名（唯一）
  email: String,         // 邮箱（唯一）
  password: String,      // 加密密码
  nickname: String,      // 昵称
  avatar: String,       // 头像URL
  role: String,         // 角色(user/admin/superadmin)
  coins: Number,        // 星币
  points: Number,       // 积分
  exp: Number,          // 经验值
  level: Number,        // 等级
  expToNextLevel: Number, // 升级所需经验
  expThisLevel: Number,  // 当前等级已获得的经验
  lastGiftDate: Date,   // 最后领取礼物日期
  createdAt: Date,
  updatedAt: Date,

  // 方法
  addExp(amount)      // 添加经验
  levelUpIfNeeded()  // 升级检查
}
```

#### 平台商店商品模型 (PlatformStoreItem)
```javascript
{

```

```
    │
```

```
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  2. 分布式锁层 (Lock Manager)   │
│  - 防止用户重复提交            │
│  - 防止竞态条件                │
│  - 自动超时清理 (30秒)         │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  3. 事务层 (Transaction)        │
│  - MongoDB 原子事务            │
│  - 全部成功或全部回滚          │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  4. 原子操作层 (Atomic $inc)   │
│  - 原子余额扣减               │
│  - 原子库存更新               │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  5. 连接池层 (Connection Pool) │
│  - 5-20个连接复用              │
│  - 心跳检测                    │
│  - 自动重连                    │
└─────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────┐
│  6. 索引层 (Database Index)    │
│  - 查询优化                    │
│  - 聚合加速                    │
└─────────────────────────────────┘
    │
    ▼
数据库操作
```

### 7.2 关键优化点

| 优化点 | 实现方式 | 问题解决 |
|------|---------|---------|
| 连接池 | MongoDB 连接池配置 | 避免连接数爆炸 |
| 分布式锁 | MongoDB唯一索引 + TTL | 防止重复提交 |
| 事务 | MongoDB 事务 | 保证数据一致性 |
| 原子操作 | $inc 操作符 | 避免竞态条件 |
| 限流 | Rate Limiter | 防止恶意请求 |
| 索引 | 复合索引/单字段索引 | 加速查询 |
| 读写关注 | majority | 保证数据安全 |
| 读写分离 | primaryPreferred | 提升读性能 |
